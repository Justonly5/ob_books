
## 安装


```BASH
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

> git 方式安装，默认会安装最新的版本。

按照完成后需要重新加载 shell：

```bash
source ~/.bashrc   # 或 source ~/.zshrc
```

### k8s 部署

```YAML

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: hermes-data-pvc
  namespace: default
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
  # 根据你的 K8s 集群环境填写存储类，例如 longhorn, nfs-client, hostpath 等
  # storageClassName: standard 

---
apiVersion: v1
kind: Service
metadata:
  name: hermes-service
  namespace: default
  labels:
    app: hermes
spec:
  ports:
    - port: 8000
      targetPort: 8000
      name: api
  selector:
    app: hermes
  type: ClusterIP

---
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: hermes-agent
  namespace: default
spec:
  serviceName: "hermes-service"
  replicas: 1 # 除非有分布式锁支持，普通模式下建议 1 副本以防长短期记忆读写冲突
  selector:
    matchLabels:
      app: hermes
  template:
    metadata:
      labels:
        app: hermes
    spec:
      containers:
        - name: hermes
          image: nousresearch/hermes-agent:v0.16.0 # 建议锁定具体版本
          imagePullPolicy: IfNotPresent
          ports:
            - containerPort: 8000
              name: api
          volumeMounts:
            - name: data-volume
              mountPath: /opt/data # 对应容器内的数据挂载点
          env:
            - name: HERMES_UID
              value: "10000"
            - name: HERMES_GID
              value: "10000"
            # 如果不使用官方 Nous Portal 登录，可以在这里注入你的商业/开源大模型密钥
            # - name: OPENROUTER_API_KEY
            #   valueFrom:
            #     secretKeyRef:
            #       name: ai-api-secrets
            #       key: openrouter-key
          resources:
            limits:
              cpu: "2"
              memory: 4Gi
            requests:
              cpu: "500m"
              memory: 1Gi
      volumes:
        - name: data-volume
          persistentVolumeClaim:
            claimName: hermes-data-pvc
```
