
## 安装


```BASH
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

> git 方式安装，默认会安装最新的版本。

按照完成后需要重新加载 shell：

```bash
source ~/.bashrc   # 或 source ~/.zshrc
```

### docker 启动


```BASH
docker run -d \
  --name hermes \
  --restart unless-stopped \
  -v ~/.hermes:/opt/data \
  -p 28642:8642 \
  
  -e API_SERVER_ENABLED=true \
  -e API_SERVER_HOST=0.0.0.0 \
  -e API_SERVER_KEY="$(openssl rand -hex 32)" \
  -e API_SERVER_CORS_ORIGINS='*' \
  nousresearch/hermes-agent gateway run
```

```BASH
docker run -d \
  --name hermes \
  --restart unless-stopped \
  -v /Users/Apple/Documents/tmp/hermes:/opt/data \
  -p 28642:8642 \
  -p 29119:9119 \
  -e HERMES_DASHBOARD=1 \
  nousresearch/hermes-agent gateway run
```

Tips：
需要配置 basic auth ，即配置用户、密码。不然没法启动 dashboard。
Dashboard 进程启动后试图绑定 0.0.0.0:9119（公网地址），但因为没有配置认证方式，被安全机制拒绝了
docker 启动时，如果未指定 HERMES_DASHBOARD_HOST 默认为 0.0.0.0，不然只有容器里可以访问。

配置方式：
进入容器 执行 python -c "from plugins.dashboard_auth.basic import hash_password; print(hash_password('你的密码'))" 生成密码 hash值，然后配置在 config.yaml
```YAML
basic_auth:                                       
 username: admin
 password_hash: 'scrypt$16384$8$1$6OsF3FuOqq10DQyKi72Q5A==$93qrzGxgg0r/XZn746qtJi3EVbXY29UGyW+v9y+XMlU='
```

### k8s 部署

```YAML

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: hermes-data-pvc
  namespace: default
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
  # 根据你的 K8s 集群环境填写存储类，例如 longhorn, nfs-client, hostpath 等
  # storageClassName: standard 

---
apiVersion: v1
kind: Service
metadata:
  name: hermes-service
  namespace: default
  labels:
    app: hermes
spec:
  ports:
    - port: 8000
      targetPort: 8000
      name: api
  selector:
    app: hermes
  type: ClusterIP

---
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: hermes-agent
  namespace: default
spec:
  serviceName: "hermes-service"
  replicas: 1 # 除非有分布式锁支持，普通模式下建议 1 副本以防长短期记忆读写冲突
  selector:
    matchLabels:
      app: hermes
  template:
    metadata:
      labels:
        app: hermes
    spec:
      containers:
        - name: hermes
          image: nousresearch/hermes-agent:v0.16.0 # 建议锁定具体版本
          imagePullPolicy: IfNotPresent
          ports:
            - containerPort: 8000
              name: api
          volumeMounts:
            - name: data-volume
              mountPath: /opt/data # 对应容器内的数据挂载点
          env:
            - name: HERMES_UID
              value: "10000"
            - name: HERMES_GID
              value: "10000"
            # 如果不使用官方 Nous Portal 登录，可以在这里注入你的商业/开源大模型密钥
            # - name: OPENROUTER_API_KEY
            #   valueFrom:
            #     secretKeyRef:
            #       name: ai-api-secrets
            #       key: openrouter-key
          resources:
            limits:
              cpu: "2"
              memory: 4Gi
            requests:
              cpu: "500m"
              memory: 1Gi
      volumes:
        - name: data-volume
          persistentVolumeClaim:
            claimName: hermes-data-pvc
```
