# Hermes 插件开发指南

## 概述

Hermes 插件系统允许在核心之外扩展功能。插件可以注册生命周期 Hook（观察/采集数据）、注册自定义工具（让 agent 调用）、注册命令等。

插件存放在 `~/.hermes/plugins/` 目录下，按分类组织目录结构。

---

## 目录结构

```
~/.hermes/plugins/<category>/<plugin-name>/
├── plugin.yaml          # 插件声明（必须）
└── __init__.py          # 主逻辑（必须提供 register(ctx) 函数）
```

示例 — ES 上报插件：

```
~/.hermes/plugins/observability/es-reporter/
├── plugin.yaml
└── __init__.py
```

---

## plugin.yaml — 插件声明

```yaml
name: es-reporter
version: "1.0.0"
description: "上报 Hermes 运行指标到 Elasticsearch"
author: "你的名字"
requires_env:
  - ES_HOST
  - ES_API_KEY
hooks:
  - pre_api_request
  - post_api_request
  - pre_tool_call
  - post_tool_call
  - on_session_start
  - on_session_end
```

| 字段 | 说明 |
|------|------|
| `name` | 插件名，唯一标识 |
| `version` | 版本号 |
| `description` | 描述 |
| `author` | 作者 |
| `requires_env` | 需要的环境变量（在 `.env` 中配置），缺失时插件不生效 |
| `hooks` | 声明要注册的 Hook 列表（声明式，便于管理） |

## 可用 Hook 列表

>https://hermes-agent.nousresearch.com/docs/zh-Hans/user-guide/features/hooks#plugin-hooks


从源码 `hermes_cli/plugins.py` 的 `VALID_HOOKS` 中定义：

| Hook | 触发时机 | 回调参数（部分） | 适合采集 |
|------|----------|-----------------|---------|
| `pre_llm_call` | LLM 调用前（旧版，按 turn） | task_id, session_id, model, provider, messages | 用户输入 |
| `post_llm_call` | LLM 调用后（旧版，按 turn） | task_id, session_id, model, provider, assistant_message | 模型回复 |
| `pre_api_request` | **每次 LLM API 请求前** | task_id, session_id, model, provider, api_call_count, message_count, approx_input_tokens, turn_id, api_request_id | 请求元信息 |
| `post_api_request` | **每次 LLM API 请求后** | task_id, session_id, model, provider, api_call_count, api_duration, finish_reason, usage, response, turn_id | Token 用量、耗时、费用 |
| `api_request_error` | API 请求出错 | task_id, session_id, model, provider, error, turn_id | 错误信息 |
| `pre_tool_call` | **工具调用前** | tool_name, args, task_id, session_id, tool_call_id, turn_id | 工具名、参数 |
| `post_tool_call` | **工具调用后** | tool_name, args, result, task_id, session_id, tool_call_id, turn_id | 工具返回结果 |
| `on_session_start` | 会话开始 | session_id, platform, model, provider | 会话创建 |
| `on_session_end` | 会话结束 | session_id, platform, model, provider, turn_count, api_call_count, duration_s | 会话汇总 |
| `subagent_start` | 子代理启动 | — | 子任务信息 |
| `subagent_stop` | 子代理结束 | — | 子任务结果 |
| `pre_gateway_dispatch` | Gateway 消息分发前 | event, gateway, session_store | 可拦截/改写消息 |
| `pre_approval_request` | 危险命令审批前 | command, description, session_key | 审批审计 |
| `post_approval_response` | 危险命令审批后 | command, choice | 审批结果审计 |
| `transform_llm_output` | LLM 输出转换 | — | 可替换回复文本 |
| `transform_tool_result` | 工具结果转换 | — | 可修改工具返回 |
| `transform_terminal_output` | 终端输出转换 | — | 可修改终端输出 |

## `__init__.py` 骨架

```python
"""es-reporter — Hermes 插件：上报运行指标到 Elasticsearch"""

import json
import logging
import os
from typing import Any, Optional

logger = logging.getLogger(__name__)


# ── 工具函数 ──────────────────────────────────────────

def _env(name: str, default: str = "") -> str:
    return os.environ.get(name, default).strip()


# ── 客户端初始化（惰性加载） ──────────────────────────

_CLIENT = None
_INIT_FAILED = object()

def _get_client() -> Optional[Any]:
    global _CLIENT
    if _CLIENT is _INIT_FAILED:
        return None
    if _CLIENT is not None:
        return _CLIENT

    host = _env("ES_HOST")
    api_key = _env("ES_API_KEY")
    if not host or not api_key:
        logger.warning("ES Reporter: ES_HOST or ES_API_KEY not set — plugin is inert.")
        _CLIENT = _INIT_FAILED
        return None

    try:
        import httpx
        _CLIENT = {
            "host": host.rstrip("/"),
            "api_key": api_key,
            "client": httpx.Client(timeout=10.0),
        }
    except Exception as exc:
        logger.warning("ES Reporter: init failed: %s", exc)
        _CLIENT = _INIT_FAILED
        return None

    return _CLIENT


def _send(index: str, body: dict) -> None:
    """发送一条文档到目标存储"""
    client = _get_client()
    if client is None:
        return

    from datetime import datetime, timezone
    body["@timestamp"] = datetime.now(timezone.utc).isoformat()

    url = f"{client['host']}/{index}/_doc"
    try:
        resp = client["client"].post(
            url,
            json=body,
            headers={
                "Content-Type": "application/json",
                "Authorization": f"ApiKey {client['api_key']}",
            },
        )
        if resp.status_code not in (200, 201):
            logger.warning("ES write failed: HTTP %s", resp.status_code)
    except Exception as exc:
        logger.warning("ES request failed: %s", exc)


# ── Hook 处理函数 ─────────────────────────────────────

def on_pre_api_request(*, task_id="", session_id="", platform="",
                       model="", provider="", api_call_count=0,
                       message_count=0, approx_input_tokens=0,
                       turn_id="", **_) -> None:
    _send("hermes-api-request", {
        "event": "api_request_start",
        "session_id": session_id,
        "task_id": task_id,
        "turn_id": turn_id,
        "model": model,
        "provider": provider,
        "platform": platform,
        "api_call_count": api_call_count,
        "message_count": message_count,
        "approx_input_tokens": approx_input_tokens,
    })


def on_post_api_request(*, task_id="", session_id="", provider="",
                        model="", api_call_count=0, response=None,
                        api_duration=0.0, finish_reason="",
                        usage=None, turn_id="", **_) -> None:
    usage_details = {}
    if isinstance(usage, dict):
        usage_details = {
            "input_tokens": usage.get("input_tokens", 0),
            "output_tokens": usage.get("output_tokens", 0),
        }

    _send("hermes-api-request", {
        "event": "api_request_complete",
        "session_id": session_id,
        "task_id": task_id,
        "turn_id": turn_id,
        "model": model,
        "provider": provider,
        "api_call_count": api_call_count,
        "api_duration_s": round(api_duration, 3),
        "finish_reason": finish_reason,
        "usage": usage_details,
    })


def on_pre_tool_call(*, tool_name="", args=None, task_id="",
                     session_id="", tool_call_id="",
                     turn_id="", **_) -> None:
    _send("hermes-tool-call", {
        "event": "tool_call_start",
        "session_id": session_id,
        "task_id": task_id,
        "turn_id": turn_id,
        "tool_name": tool_name,
        "tool_call_id": tool_call_id,
        "args": str(args)[:500] if args else "",
    })


def on_post_tool_call(*, tool_name="", args=None, result=None,
                      task_id="", session_id="", tool_call_id="",
                      turn_id="", **_) -> None:
    _send("hermes-tool-call", {
        "event": "tool_call_complete",
        "session_id": session_id,
        "task_id": task_id,
        "turn_id": turn_id,
        "tool_name": tool_name,
        "tool_call_id": tool_call_id,
        "result": str(result)[:500] if result else "",
    })


def on_session_start(*, session_id="", platform="",
                     model="", provider="", **_) -> None:
    _send("hermes-session", {
        "event": "session_start",
        "session_id": session_id,
        "platform": platform,
        "model": model,
        "provider": provider,
    })


# ── 注册入口（必须） ──────────────────────────────────

def register(ctx) -> None:
    """Hermes 插件系统自动调用此函数"""
    ctx.register_hook("pre_api_request", on_pre_api_request)
    ctx.register_hook("post_api_request", on_post_api_request)
    ctx.register_hook("pre_tool_call", on_pre_tool_call)
    ctx.register_hook("post_tool_call", on_post_tool_call)
    ctx.register_hook("on_session_start", on_session_start)
```

## 注册自定义工具

除了 Hook，插件还可以注册工具，让 agent 在对话中主动调用：

```python
def my_tool(param: str) -> str:
    """工具处理函数，必须返回 JSON 字符串"""
    return json.dumps({"result": f"processed: {param}"})

def register(ctx) -> None:
    ctx.register_tool(
        name="my_tool",
        description="描述这个工具做什么",
        parameters={
            "type": "object",
            "properties": {
                "param": {
                    "type": "string",
                    "description": "参数说明"
                }
            },
            "required": ["param"]
        },
        handler=lambda args, **kw: my_tool(args.get("param", "")),
    )
```

## 启用插件

### 方式一：通过 hermes tools 交互式启用

```bash
hermes tools
```

### 方式二：直接修改 config.yaml

```bash
hermes config set plugins.enabled '["browser-browser-use", "observability/es-reporter"]'
```

### 方式三：编辑 config.yaml

```yaml
plugins:
  enabled:
    - browser-browser-use
    - observability/langfuse
    - observability/es-reporter
  disabled: []
```

### 配置环境变量

在 `~/.hermes/.env` 中添加：

```
ES_HOST=https://your-es-cluster:9200
ES_API_KEY=base64...apikey
```

### 重启生效

修改后需要重启 Hermes（重启 Desktop 或 `/restart`）使插件加载。

## 插件开发最佳实践

### 1. 惰性初始化客户端

```python
_CLIENT = None
_INIT_FAILED = object()

def _get_client():
    global _CLIENT
    if _CLIENT is _INIT_FAILED:
        return None
    if _CLIENT is not None:
        return _CLIENT
    # 首次初始化...
```

- 第一次调用 Hook 时才初始化
- 初始化失败后缓存 `_INIT_FAILED`，避免每次调用都重试
- 这样即使配置缺失，插件也不会报错，只是安静地不工作

### 2. 数据安全

- 截断长字段（建议 5000 字符以内）
- 不要发送 base64 图片/音频等大 payload
- 嵌套深度限制（建议 4 层以内）
- 数组/字典元素数限制（建议 30 以内）

### 3. 不要阻塞主流程

- Hook 回调是同步调用的，不要在 Hook 中做耗时操作
- 如果目标存储响应慢，考虑异步发送或使用队列

### 4. 使用 `**_: Any` 兜底

Hook 回调参数会随 Hermes 版本变化，用 `**_: Any` 接收未来可能新增的参数，避免签名不匹配导致报错。

### 5. 日志规范

- 用 `logger.info/warning` 记录关键事件
- 用环境变量控制调试日志开关（如 `ES_DEBUG=true`）
- 不要用 `print()`

## 参考：es-reporter 插件（完整实现）

位置：`~/.hermes/plugins/observability/es-reporter/`

采集指标：

| ES 索引 | 事件 | 内容 |
|---------|------|------|
| `hermes-api-request` | `api_request_start` | LLM 请求前：模型、消息数、预估 token |
| `hermes-api-request` | `api_request_complete` | LLM 请求后：token 用量、耗时、费用 |
| `hermes-api-error` | `api_request_error` | API 请求失败 |
| `hermes-tool-call` | `tool_call_start/complete` | 工具调用：工具名、参数、结果 |
| `hermes-session` | `session_start/end` | 会话生命周期 |

环境变量：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `ES_HOST` | ES 地址（必填） | — |
| `ES_API_KEY` | ES API Key（必填） | — |
| `ES_INDEX_PREFIX` | 索引前缀 | `hermes` |
| `ES_MAX_CHARS` | 单字段最大字符 | `5000` |
| `ES_DEBUG` | 调试日志 | `false` |

## 参考：Langfuse 插件（官方示例）

位置：`~/.hermes/hermes-agent/plugins/observability/langfuse/`

Langfuse 插件是官方自带的 observability 插件，功能更完整：
- Trace 生命周期管理（root trace → generation → tool 三层嵌套）
- Token 用量和费用估算
- read_file 结果智能摘要
- base64 data URI 自动脱敏
- 并发 trace 状态管理（256 上限）
- 过期 trace 自动淘汰

可以直接作为开发参考模板。
