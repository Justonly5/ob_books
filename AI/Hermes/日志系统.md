# Hermes 日志系统

## 日志文件一览

所有日志文件位于 `~/.hermes/logs/`：

| 文件                      | 级别       | 大小/滚动      | 内容                                         |
| ----------------------- | -------- | ---------- | ------------------------------------------ |
| **`agent.log`**         | INFO+    | 5MB × 3 份  | **主日志**：所有 agent/tool/session 活动           |
| **`errors.log`**        | WARNING+ | 2MB × 2 份  | 错误和警告，快速排查                                 |
| **`gateway.log`**       | INFO+    | 5MB × 3 份  | Gateway 组件日志（platform adapter、session 管理等） |
| **`gateway.error.log`** | —        | 无滚动        | Gateway 进程的 stderr 重定向                     |
| **`gui.log`**           | INFO+    | 10MB × 5 份 | Dashboard/WebSocket/TUI-gateway 日志         |
| **`desktop.log`**       | —        | —          | Desktop 应用日志                               |
| **`bootstrap-*.log`**   | —        | —          | 安装/更新引导日志                                  |

## 核心代码

`~/.hermes/hermes-agent/hermes_logging.py`

### `setup_logging()` 函数

所有入口（CLI、Gateway、GUI）启动时调用，负责初始化日志系统。

**参数：**

| 参数 | 说明 |
|------|------|
| `mode` | 调用上下文：`"cli"`、`"gateway"`、`"gui"`、`"cron"` |
| `log_level` | 最低级别，默认从 `config.yaml` 读取 |
| `max_size_mb` | 滚动大小，默认从 `config.yaml` 读取 |
| `backup_count` | 备份数，默认从 `config.yaml` 读取 |
| `force` | 强制重新初始化 |

### 调用入口

| 入口 | 代码位置 | 调用方式 |
|------|----------|----------|
| CLI 交互模式 | `cli.py:728` | `setup_logging(mode="cli")` |
| CLI 命令模式 | `hermes_cli/main.py:570` | `setup_logging(mode="cli")` |
| Gateway | `hermes_cli/main.py` | `setup_logging(mode="gateway")` |
| GUI (Desktop) | `hermes_cli/main.py:5575` | `setup_logging(mode="gui")` |

## 日志配置（`config.yaml`）

```yaml
logging:
  level: INFO          # agent.log 的最低级别：DEBUG / INFO / WARNING
  max_size_mb: 5       # 每个日志文件滚动大小（MB）
  backup_count: 3      # 保留的滚动备份数
```

当前默认值：5MB 滚动，保留 3 个备份。

## 日志格式

```
# 普通格式
2026-06-29 20:58:14,020 INFO [20260623_155506_af24b4] agent.turn_context: conversation turn: ...

# 格式模板（代码中硬编码）
%(asctime)s %(levelname)s%(session_tag)s %(name)s: %(message)s

# 详细格式（--verbose 模式）
%(asctime)s - %(name)s - %(levelname)s%(session_tag)s - %(message)s
```

| 占位符 | 说明 |
|--------|------|
| `%(asctime)s` | 时间戳 |
| `%(levelname)s` | 日志级别 |
| `%(session_tag)s` | 会话 ID（如 `[20260623_155506_af24b4]`），有会话上下文时显示 |
| `%(name)s` | Logger 名称（如 `agent.turn_context`、`hermes_cli.plugins`） |
| `%(message)s` | 日志内容 |

所有日志经过 `RedactingFormatter`（`agent/redact.py`），**自动脱敏**（API key、token 等不会写入磁盘）。

## 滚动机制

使用 Python 标准库 `RotatingFileHandler`（POSIX）/ `ConcurrentRotatingFileHandler`（Windows）：

```
agent.log        → 当前日志
agent.log.1      → 第一次滚动
agent.log.2      → 第二次滚动
agent.log.3      → 第三次滚动（最旧，再滚动就覆盖）
```

`_ManagedRotatingFileHandler`（增强版）额外功能：
- **检测外部滚动**（如 `logrotate` 命令手动切割），自动重新打开新文件
- 在 managed 模式下自动设置 `chmod 0660` 组权限

## 组件过滤

`gateway.log` 和 `gui.log` 使用 `_ComponentFilter` 只记录特定组件的日志：

| 组件 | Logger 前缀 |
|------|-------------|
| **gateway** | `gateway.*`, `hermes_plugins.*`, `plugins.platforms.*` |
| **agent** | `agent.*`, `run_agent`, `model_tools` |
| **tools** | `tools.*` |
| **cli** | `hermes_cli.*`, `cli` |
| **cron** | `cron.*` |
| **gui** | `hermes_cli.web_server`, `hermes_cli.pty_bridge`, `tui_gateway.*`, `uvicorn.*` |

## 会话关联

每条日志自动注入当前会话 ID，便于过滤和关联：

```python
from hermes_logging import set_session_context, clear_session_context

set_session_context("20260623_155506_af24b4")
# 之后所有日志自动带上 [20260623_155506_af24b4]

clear_session_context()  # 会话结束时清除
```

## 控制台输出

### 方式一：`--verbose` 模式（已有功能）

```bash
hermes -v                # CLI 模式
hermes chat -v -q "..."  # 单次查询
```

- 在 stderr 上添加 `StreamHandler`，输出 **DEBUG 级别** 日志
- 格式更详细：`%(asctime)s - %(name)s - %(levelname)s - %(message)s`
- 仅在 CLI 交互模式生效，Gateway 和 GUI 模式不支持

### 方式二：`logging.console: true`（需改代码）

在 `config.yaml` 中添加：

```yaml
logging:
  level: INFO
  max_size_mb: 5
  backup_count: 3
  console: true          # 同时输出到 stdout，用于 K8s/ELK 采集
```

需要在 `hermes_logging.py` 的 `setup_logging()` 中添加对应的 `StreamHandler`。

## 与 Java 日志体系对比

| 能力 | Java (logback) | Hermes |
|------|---------------|--------|
| 配置文件格式 | XML | **YAML**（`config.yaml` 的 `logging:` 段） |
| 输出格式 | 可自定义 pattern | 硬编码，不支持自定义 |
| 日志目录 | 可配置 | 固定 `~/.hermes/logs/` |
| 滚动策略 | 时间/大小/组合 | **仅大小滚动**（`max_size_mb` + `backup_count`） |
| 按时间滚动 | 支持 `TimeBasedRollingPolicy` | ❌ 不支持 |
| 异步日志 | 支持 `AsyncAppender` | ❌ 不支持 |
| 按 Logger 控制级别 | 支持 | 硬编码了第三方库的抑制，不支持用户自定义 |
| 多 Appender | 支持 | 固定 4 个文件 handler + 1 个 stderr handler |
| 日志脱敏 | 需自实现 | ✅ `RedactingFormatter` 自动脱敏 |

## K8s 生产部署日志采集方案

### 方案 A：改代码加 `console: true`

在 `hermes_logging.py` 中添加 stdout handler，由 `config.yaml` 控制开关。

### 方案 B：`--verbose` 模式（已有，无需改代码）

```bash
hermes -v
```

但只输出 DEBUG 级别，且仅在 CLI 模式生效。

### 方案 C：外部 `tail -F` + sidecar

不改代码，在 K8s Pod 中添加 sidecar 容器 tail 日志文件：

```yaml
containers:
  - name: hermes
    image: nousresearch/hermes-agent
    # ...
  - name: log-collector
    image: alpine
    command: ["tail", "-F", "/root/.hermes/logs/agent.log"]
    volumeMounts:
      - name: hermes-data
        mountPath: /root/.hermes
```

### 方案 D：用 `logrotate` 做时间维度切割

```bash
# /etc/logrotate.d/hermes
/Users/Apple/.hermes/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    copytruncate
}
```

Hermes 的 `_ManagedRotatingFileHandler` 能自动检测外部滚动。

### 推荐：方案 A（改代码）

```
Hermes Agent (Pod)
  ├── stdout ──→ containerd ──→ /var/log/pods/*.log
  │                              ↓
  │                         Fluent Bit / Filebeat (DaemonSet)
  │                              ↓
  │                         Elasticsearch ← Kibana
  │
  └── stderr ──→ (错误日志同样链路)
```

Fluent Bit 配置示例：

```yaml
[INPUT]
    name              tail
    path              /var/log/containers/hermes-*.log
    multiline.parser  docker, cri
    tag               hermes.*

[FILTER]
    name              parser
    match             hermes.*
    parser            hermes-log

[PARSER]
    name              hermes-log
    format            regex
    regex             ^(?<timestamp>\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2},\d{3}) (?<level>\w+)(?: \[(?<session_id>[^\]]+)\])? (?<logger>[^:]+): (?<message>.*)$

[OUTPUT]
    name              es
    match             hermes.*
    host              ${ES_HOST}
    port              9200
    index             hermes-logs-${%Y.%m.%d}
```
