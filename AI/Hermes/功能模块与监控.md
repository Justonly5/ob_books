# Hermes 功能模块与监控

> 2026-06-25 整理

## 一、功能模块总览

| 模块            | 说明                                      | 是否需要单独启动  |
| ------------- | --------------------------------------- | :-------: |
| **CLI**       | 命令行交互界面，核心入口                            |   ✅ 必须    |
| **Gateway**   | 多平台消息中枢（Telegram/飞书/钉钉/微信等 20+）         |    按需     |
| **Cron**      | 定时任务调度器（自然语言/cron 表达式）                  | 随 Gateway |
| **Kanban**    | 多 Profile/多 Worker 协作看板（SQLite）         |    按需     |
| **Skills**    | 可复用的过程性知识库（SKILL.md）                    |   自动加载    |
| **Memory**    | 跨会话持久记忆（内置 SQLite / Honcho / Mem0）      |   自动加载    |
| **Toolsets**  | 模块化工具开关（web/terminal/file/vision 等 20+） |   按需启用    |
| **Profiles**  | 多套独立配置/会话/技能/记忆                         |    按需     |
| **Plugins**   | 插件扩展                                    |    按需     |
| **MCP**       | Model Context Protocol，双向工具扩展           |    按需     |
| **Webhooks**  | HTTP Webhook 事件驱动                       | 随 Gateway |
| **Sessions**  | 会话管理（SQLite + FTS5 全文搜索）                |    自动     |
| **Voice**     | STT（语音→文字）+ TTS（文字→语音）                  |    按需     |
| **Dashboard** | 无独立 Web Dashboard，通过 CLI 查看             |     -     |

### 核心启动模式

```bash
hermes                    # CLI 交互（最基础，零额外配置）
hermes gateway run        # 网关服务（7×24h 常驻后台）
hermes gateway install    # 安装为 systemd 后台服务
```

其他模块不需要单独启动进程，内嵌在 CLI 或 Gateway 中。

## 二、监控方案

### 方案 1：agent.log（零配置，推荐日常使用）

```bash
tail -f ~/.hermes/logs/agent.log
```

记录内容：

| 记录类型 | 示例 |
|---------|------|
| LLM 调用 | `API call #1: model=deepseek-v4-pro in=18671 out=114 latency=8.5s cache=22%` |
| 工具调用 | `tool search_files completed (0.27s, 3835 chars)` |
| Turn 结束 | `Turn ended: reason=text_response api_calls=4/150 tool_turns=3` |
| API 失败 | `API call failed (attempt 1/3) error_type=NotFoundError` |

### 方案 2：hermes insights（统计看板）

```bash
hermes insights --days 7
```

输出：Token 用量、模型分布、工具调用 Top、活跃模式等。

### 方案 3：request_dump（失败请求快照）

位置：`~/.hermes/sessions/request_dump_*.json`

当 API 调用失败时，保存完整请求体（含 system prompt、工具列表）。

### 方案 4：Langfuse（全链路追踪，推荐生产环境）

开源 LLM 可观测平台，提供 Web 可视化看板。

**配置方式：**

```bash
# 写入 ~/.hermes/.env
export HERMES_LANGFUSE_PUBLIC_KEY="pk-lf-xxx"
export HERMES_LANGFUSE_SECRET_KEY="***"
export HERMES_LANGFUSE_BASE_URL="https://us.cloud.langfuse.com"
```

然后 `/reload` 或重启 Hermes 生效。

**Langfuse 能看什么：**

```
Trace: session_xxx
├── LLM Call: deepseek-v4-flash (8.5s, 18,671 in / 114 out)
│   ├── Input: [完整 prompt]
│   └── Output: [模型回复]
├── Tool Call: terminal (0.3s)
│   ├── Input: "ps aux | grep hermes"
│   └── Output: "PID 5115 26.6%..."
└── ...
```

## 三、系统资源监控

```bash
# 进程资源
ps aux | grep -i hermes

# 磁盘占用
du -sh ~/.hermes

# 日志大小
du -sh ~/.hermes/logs
```

### 当前环境参考值

| 进程 | CPU | 内存 |
|------|:---:|:----:|
| dashboard (Python) | ~26% | ~233MB |
| TUI (Node.js) | ~10% | ~144MB |
| gateway (Python) | ~0.1% | ~186MB |
